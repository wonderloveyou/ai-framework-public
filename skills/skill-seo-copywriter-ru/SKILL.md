# SEO Copywriter RU — Skill

## Goal
Написать текст, который одновременно:
1. Отвечает на запрос пользователя (search intent)
2. Нравится поисковым системам (ключевые слова, структура)
3. Нравится AI-поиску (GEO: Perplexity, SGE, Bing Copilot)
4. Решает бизнес-задачу (конверсия, трафик, экспертиза)

## Inputs
- Тема или список ключевых слов
- Search intent: информационный / коммерческий / навигационный / транзакционный
- Целевая аудитория
- Цель статьи / бизнес-задача
- Опционально: конкурентные статьи (URL), тональность, STYLE.md

## Outputs
- Текст с SEO-разметкой (H1-H3, keyphrase, LSI, internal links)
- Мета-данные: Title, Description, Slug
- Чеклист соответствия требованиям

## Workflow

### Phase 0: Определи бизнес-задачу
Прежде чем писать — ответь:
1. Кто будет читать? (роль, контекст, проблема)
2. Что они должны сделать после текста?
3. Какая метрика успеха? (трафик, конверсия, позиции, линки)
4. Как мы измерим, что текст сработал?

### Phase 1: Сбор семантики
1. Определи фокусное ключевое слово (main keyphrase). Проверь частотность через Wordstat: запросы `(+купить|+цена|+доставка) [тема]` для коммерции, `(как|почему|что такое) [тема]` для инфо
2. Собери LSI-фразы и близкие по смыслу запросы. Проанализируй топ-5 конкурентов через Keys.so / Mutagen
3. Определи частотность: 0.5-1.5% для основного ключа, LSI — естественно
4. Проанализируй топ-5 конкурентов в выдаче на предмет:
   - Структуры (какие H2/H3 они используют?)
   - Объёма (сколько слов?)
   - Интента (на какой вопрос отвечают?)
   - Тона (официально / экспертно / популярно)
5. Проверь People Also Ask (блок «Люди также спрашивают») — из него берутся подвопросы для структуры

### Phase 2: Интент и структура
1. Определи search intent:
   - Информационный: ответ на вопрос, гайд, обзор
   - Коммерческий: сравнение, выбор, "лучший", отзывы
   - Навигационный: "как сделать X в Y"
   - Транзакционный: цена, купить, заказать
2. Составь структуру (H1 → H2 → H3)
3. Каждый H2 раздел отвечает на один подвопрос пользователя
4. Для GEO (AI-поиск): текст должен быть понятен без контекста — AI-поисковик может вырвать любой абзац

### Phase 3: Построение структуры (Outline)
1. **Главный принцип**: сначала интент, потом структура, потом текст
2. **H1**: содержит main keyphrase, привлекает внимание (заголовок, который читают)
3. **H2-H4**: иерархия с включением ключевых слов и LSI. H2 — структурные блоки, H3 — детали
4. **Блок FAQ**: включи 3-5 вопросов из «Люди также спрашивают» — они дают место в Google
5. **Обрати внимание на featured snippets**: если в топе есть списки, таблицы или короткие ответы — делай такой же формат
6. **Каждый H2 раздел отвечает на один подвопрос пользователя**
7. **Для GEO (AI-поиск)**: любой абзац должен быть понятен вне контекста — AI-поисковик может вырвать его

### Phase 4: Написание
**SEO-требования:**
- **Введение**: первые 100 слов содержат keyphrase, отвечают на «ну и что?»
- **Плотность**: основной ключ 0.5-1.5%, LSI — естественно, равномерно по тексту
- **Internal links**: 3-5 ссылок на релевантные статьи сайта. Якорные тексты — разные, не повторять один keyphrase
- **External links**: на авторитетные источники первого уровня (офиц. документы, исследования, отраслевые отчёты)
- **Изображения**: минимум 1 на 800 слов, с заполненным Alt, содержащим keyphrase
- **Читаемость**: предложения до 20 слов, абзацы 2-4 предложения, используй списки и таблицы
- **Инфостиль**: никакого канцелярита, факты вместо оценок
- **CTA**: в конце статьи — что делать читателю дальше

**Источники по приоритету:**
- Уровень 1: официальные документы, peer-review, отраслевые базы данных
- Уровень 2: экспертные статьи в профильных медиа, кейсы с цифрами
- Уровень 3: блоги, форумы, пользовательский контент (только как дополнение)

### Phase 5: Мета-данные
- **Title**: до 60 символов, keyphrase в начале, CTA/выгода. Уникальный, не дублируется с конкурентами
- **Meta description**: 140-155 символов, keyphrase, суть + CTA. Должен «продавать» клик
- **Slug**: короткий, содержит keyphrase, без стоп-слов (site.com/blog/kak-vybrat-pylesos)
- **Alt тексты**: для изображений с keyphrase (если есть)

### Phase 6: Финальный чеклист
**Основное:**
- [ ] Объём соответствует ТЗ
- [ ] Уникальность текста ≥ 95%
- [ ] Текст соответствует search intent
- [ ] Статья даёт исчерпывающий ответ на запрос

**SEO:**
- [ ] H1 содержит main keyphrase, привлекает внимание
- [ ] Keyphrase в первом абзаце
- [ ] Keyphrase в 2-3 H2/H3
- [ ] Плотность keyphrase 0.5-1.5%
- [ ] LSI-фразы распределены равномерно
- [ ] URL (slug) содержит keyphrase

**Читаемость:**
- [ ] Предложения до 20 слов
- [ ] Абзацы 2-4 предложения
- [ ] Есть списки и/или таблицы
- [ ] Нет канцелярита и AI-штампов

**Ссылки и медиа:**
- [ ] 3-5 internal links с разными анкорами
- [ ] External links на авторитетные источники
- [ ] Изображения с заполненным Alt
- [ ] FAQ-блок с вопросами из «Люди также спрашивают»

**GEO-готовность:**
- [ ] Любой абзац понятен вне контекста
- [ ] Есть таблицы/списки для featured snippets
- [ ] Текст содержит ответы на прямые вопросы (кто, что, как, почему, сколько)

## Hard rules
- Не жертвуй читаемостью ради SEO — текст для людей, не для роботов
- Ключевые слова вставляй естественно, без переспама
- Не выдумывай факты, цифры, даты
- Не используй канцелярит и AI-штампы
- Для коммерческих запросов — конкретные характеристики, цены, сравнения
- Для информационных — глубина, примеры, структура

## Пример

### Title
Плохо: «SEO-продвижение сайта — услуги и цены»
Хорошо: «SEO-продвижение сайта: цена 15 000 ₽, срок — 3 месяца»

### Структура (коммерческий запрос "купить холодильник")
H1: Купить холодильник [бренд/тип] в Москве — цена от X ₽
H2: Какие холодильники подходят для [контекст]?
H2: Сравнение моделей: [модель A] vs [модель B]
H2: Доставка и установка
H3: Сколько стоит доставка?
H3: Как подготовить место?
H2: Частые вопросы
H2: Отзывы покупателей
