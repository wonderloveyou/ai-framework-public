# Session Log

## Goal
Сохранить максимально подробный лог текущей сессии в Obsidian vault. Результат — близкая к 1:1 копия диалога с полной хронологией, всеми шагами, решениями и контекстом. Токены не экономим — это для восстановления контекста при следующей сессии.

## Workflow

### Phase 1: Определи контекст
1. Проект — из рабочей директории или первых сообщений диалога
2. Тема сессии — ёмкое описание в 3-7 слов
3. Дата — сегодняшняя

### Phase 2: Восстанови хронологию
Пройди по ВСЕМ сообщениям диалога и для каждого шага зафиксируй:

**Что включать:**
- Полный текст или близкий пересказ сообщений пользователя
- Твои tool calls: какой инструмент, параметры (ключевые), результат
- Принятые решения и почему
- Изменённые/созданные файлы и что именно изменилось
- Ошибки и как исправлены
- Промежуточные проверки (тесты, линтер, замеры строк)

**Как писать:**
- Не сокращай. Пиши детально, близко к исходному диалогу
- Сохраняй порядок — хронология важнее категоризации

### Phase 3: Сохрани лог в Obsidian vault
Vault root: `{OBSIDIAN_VAULT}`
Путь для записи: `Jarvis/01_raw/sessions/YYYY-MM-DD-<краткий-слаг>.md`
Инструмент: `obsidian_write_note` с mode: `overwrite`

Формат файла:
```markdown
# Сессия: <тема>
**Дата:** YYYY-MM-DD
**Проект:** <путь>
**Длительность:** оценка

## Полная хронология

### 1. <заголовок шага>
**Пользователь:** <сообщение пользователя>
**Действия:** <tool calls и что сделано>
**Результат:** <что получилось>
**Файлы:** <изменённые файлы>

### 2. <заголовок шага>
...

## Итоги
**Принятые решения:**
- решение 1
- решение 2

**Файлы (создано/изменено):**
- `path/to/file` — что изменилось

**Ошибки и фиксы:**
- ошибка → как исправили

**На следующую сессию:**
- что осталось доделать
```

### Phase 4: Обнови program.md (в корне ПРОЕКТА)
Если в проекте есть `program.md` (в корне текущей рабочей директории):
- В `## Phase History` добавь запись: `### Phase N (YYYY-MM-DD): <тема сессии>`
- Если были gotchas → в `## Gotchas & Pitfalls`
- Если были архитектурные решения → в `## Architecture Decisions`

### Phase 5: Обнови decisions.md в vault
Если были важные решения → используй `obsidian_write_note` для записи в `Jarvis/05_logs/decisions.md` (внутри vault):
```markdown
## YYYY-MM-DD: <краткое описание решения>
**Context:** ...
**Decision:** ...
```

### Phase 6: Финальное сообщение
Коротко пользователю: где сохранено, что обновлено, что на следующий раз.

## Outputs
1. Файл сессии в `Jarvis/01_raw/sessions/YYYY-MM-DD-<слаг>.md`
2. Обновлённый `program.md` (если был)
3. Обновлённый `Jarvis/05_logs/decisions.md` (если были решения)
4. Короткое confirmation в чат

## Hard rules
- Не сокращай содержимое диалога — пиши максимально подробно
- Не выдумывай то, чего не было в сессии
- Если диалог очень длинный (>30 сообщений) — группируй шаги по логическим блокам, но сохраняй детали
- Сохраняй лог сессии через `obsidian_write_note` в vault по пути `Jarvis/01_raw/sessions/YYYY-MM-DD-<слаг>.md`
- После завершения НЕ продолжай работу — сессия окончена
