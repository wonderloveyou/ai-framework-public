# Bitrix AI Agent Carousel Example

Source: `W:\0SMM\projects\velmi-ai\habr-statia-crm\article_images.html`

## Accepted Structure

1. Cover: `AI вместо менеджера`
2. Context: Bitrix24 at the center, site form, Telegram, ~400 leads/month, 4 managers, 3-second webhook constraint, FastAPI/Redis/GPT-4o/REST.
3. Problem: big `70%` and compact funnel.
4. Result/mechanism: `2-3 ч -> 30 с`.
5. Final principle: do not give CRM decisions to the model.

## Pattern Mapping

- context slide: `central-object-orbit-facts`
- problem slide: `big-metric-meaning`
- speed slide: `split-comparison`
- final slide: `split-comparison` with responsibility boundary

## What Fixed The Deck

- Context became visual, not paragraph text.
- Final slide stopped repeating metrics.
- LLM/code responsibilities were split visually.
- Overloaded panels were simplified before shrinking fonts.

## Reusable Principle

For AI automation cases:

- LLM: dialogue, intent, next question, summary.
- Code: deduplication, status whitelist, deadlines, thresholds, CRM writes.

End with an implementation boundary when the audience is technical.
