# carousel-builder

Use this skill for social media carousels, article covers, HTML card decks, batch image generation, and design QA.

## Core Workflow

1. Identify platform, dimensions, slide count, audience, tone, and brand profile.
2. Turn source material into slide roles.
3. Write short copy: hook, 1-3 bullets per inner slide, final CTA/principle.
4. Create a visual brief for each slide.
5. Render as HTML/CSS cards, image prompts, or batch templates.
6. Validate readability, density, brand consistency, export filenames, and card count.

## Slide Roles

- cover = hook/promise
- context = situation/system/stakes
- problem = pain/failure/misconception
- mechanism = what changed/how it works
- result = metric/proof/example
- final = CTA, summary, or transferable principle

## Platform Defaults

- Instagram: 1080x1350 or 1080x1080
- LinkedIn: 1080x1350, 1200x1500, or 1200x1200
- Telegram/VK cards: 1920x1920 or 1080x1080
- Article cover/explainer: 1920x1080

## Quality Gates

- one idea per slide
- one dominant visual object
- readable in mobile preview
- final slide does not duplicate previous slide
- text does not collide with fixed UI/decor
- brand colors and typography stay consistent
- export count matches filenames and buttons
